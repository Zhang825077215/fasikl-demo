Fasikl Developer Interview Assignment
Thank you for your interest in Fasikl. As part of the second-round interview, we invite shortlisted candidates to complete a short practical engineering exercise based on realistic product scenarios.
The goal is not to build a polished production system. We want to understand how you think, scope a problem, write code, make tradeoffs, and communicate decisions.
Time Expectation
Please spend approximately 3-4 hours.
A strong submission does not need to be feature-complete.
What to Submit
1.A GitHub repository or zip file with your code
2.A short README with setup/run instructions
3.A brief note (1-2 page at most or markdown) covering:
○What you built
○Key design decisions
○What you would improve next
○Any AI tools or external resources used
4.Be ready to give a short walkthrough during the interview
Choose One of the Following Problems:

Build a small physician-facing dashboard that summarizes patient status.
Core Features
●View patient list

Problem C - Patient Message AI Assistant
Build a simple patient-facing speech AI feature embedded inside the portal or mobile app. The goal is to help patients get quick answers, complete simple tasks, and reduce avoidable support calls, while allowing escalation to human support when needed.
You may use rules, mock APIs, an LLM, or a simple hybrid approach. Text transcripts are sufficient.
Core Features
Input a patient question (speech transcript or text) and return some combination of:
●Topic/category (e.g., device issue, clinical concern, billing, logistics, account/access)
●Helpful response (canned reply, FAQ snippet, or generated text)
●Action for next step (auto_reply, route_to_<team>, escalate_urgent)
Add one useful feature of your choice, for example:
●Link or navigation suggestion inside the portal/app
●Conversation history or follow-up context
